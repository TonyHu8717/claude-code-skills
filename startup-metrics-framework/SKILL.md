# 创业公司指标框架

跟踪、计算和优化不同创业公司商业模式从种子轮到 A 轮的关键绩效指标的综合指南。

## 概述

在正确的阶段跟踪正确的指标。专注于对融资和运营卓越至关重要的单位经济性、增长效率和现金管理指标。

## 通用创业指标

### 收入指标

**MRR（月经常性收入）**

```
MRR = Σ（活跃订阅 × 月价格）
```

**ARR（年经常性收入）**

```
ARR = MRR × 12
```

**增长率**

```
环比增长 =（本月 MRR - 上月 MRR）/ 上月 MRR
同比增长 =（本年 ARR - 去年 ARR）/ 去年 ARR
```

**目标基准：**

- 种子阶段：15-20% 环比增长
- A 轮：10-15% 环比增长，3-5 倍同比增长
- B 轮以上：100%+ 同比（40 法则）

### 单位经济性

**CAC（客户获取成本）**

```
CAC = 总 S&M 支出 / 获取的新客户数
```

包括：销售工资、营销支出、工具、管理费用

**LTV（生命周期价值）**

```
LTV = ARPU × 毛利率% ×（1 / 流失率）
```

简化版：

```
LTV = ARPU × 平均客户生命周期 × 毛利率%
```

**LTV:CAC 比率**

```
LTV:CAC = LTV / CAC
```

**基准：**

- LTV:CAC > 3.0 = 健康
- LTV:CAC 1.0-3.0 = 需要改进
- LTV:CAC < 1.0 = 不可持续

**CAC 回收期**

```
CAC 回收 = CAC /（ARPU × 毛利率%）
```

**基准：**

- < 12 个月 = 优秀
- 12-18 个月 = 良好
- > 24 个月 = 令人担忧

### 现金效率指标

**烧钱率**

```
月烧钱 = 月收入 - 月支出
```

负烧钱 = 亏损（典型的早期阶段）

**跑道**

```
跑道（月）= 现金余额 / 月烧钱率
```

**目标：** 始终保持 12-18 个月跑道

**烧钱倍数**

```
烧钱倍数 = 净烧钱 / 净新增 ARR
```

**基准：**

- < 1.0 = 卓越效率
- 1.0-1.5 = 良好
- 1.5-2.0 = 可接受
- > 2.0 = 低效

越低越好（花费更少生成 ARR）

## SaaS 指标

### 收入构成

**新增 MRR**
新客户 × ARPU

**扩展 MRR**
现有客户的追加销售和交叉销售

**收缩 MRR**
现有客户的降级

**流失 MRR**
丢失客户

**净新增 MRR 公式：**

```
净新增 MRR = 新增 MRR + 扩展 MRR - 收缩 MRR - 流失 MRR
```

### 留存指标

**客户留存率**

```
客户留存率 =（期末客户 - 新客户）/ 期初客户
```

**收入留存率（NDR - 净收入留存率）**

```
NDR =（期初 ARR + 扩展 - 收缩 - 流失）/ 期初 ARR
```

**基准：**

- NDR > 120% = 业界最佳
- NDR 100-120% = 良好
- NDR < 100% = 需要改进

**毛留存率**

```
毛留存率 =（期初 ARR - 流失 - 收缩）/ 期初 ARR
```

**基准：**

- > 90% = 优秀
- 85-90% = 良好
- < 85% = 令人担忧

### SaaS 专项指标

**魔法数字**

```
魔法数字 = 净新增 ARR（季度）/ S&M 支出（上一季度）
```

**基准：**

- > 0.75 = 高效，准备扩展
- 0.5-0.75 = 中等效率
- < 0.5 = 低效，暂不扩展

**40 法则**

```
40 法则 = 收入增长率% + 利润率%
```

**基准：**

- > 40% = 优秀
- 20-40% = 可接受
- < 20% = 需要改进

**示例：**
50% 增长 + (-10%) 利润率 = 40% ✓

**快速比率**

```
快速比率 =（新增 MRR + 扩展 MRR）/（流失 MRR + 收缩 MRR）
```

**基准：**

- > 4.0 = 健康增长
- 2.0-4.0 = 中等
- < 2.0 = 流失问题

## 市场指标

### GMV（商品交易总额）

**总交易量：**

```
GMV = Σ（交易价值）
```

**增长率：**

```
GMV 增长率 =（当期 GMV - 上期 GMV）/ 上期 GMV
```

**目标：** 早期 20%+ 环比

### 提成率

```
提成率 = 净收入 / GMV
```

**典型范围：**

- 支付处理商：2-3%
- 电商市场：10-20%
- 服务市场：15-25%
- 高价值 B2B：5-15%

### 市场流动性

**交易时间**
从上架到销售/匹配需要多长时间？

**成交率**
导致交易的请求百分比

**复购率**
多次交易的用户百分比

**基准：**

- 成交率 > 80% = 强流动性
- 复购率 > 60% = 强留存

### 市场平衡

**供需比率：**
跟踪供需双方的相对增长。

**警告信号：**

- 供应过多：低成交率，供应商受挫
- 需求过多：等待时间长，客户受挫

**目标：** 平衡增长（1:1 比率理想，但因模式而异）

## 消费/移动指标

### 参与指标

**DAU（日活跃用户）**
每天活跃的独立用户

**MAU（月活跃用户）**
每月活跃的独立用户

**DAU/MAU 比率**

```
DAU/MAU = DAU / MAU
```

**基准：**

- > 50% = 卓越（日常习惯）
- 20-50% = 良好
- < 20% = 参与度弱

**会话频率**
每用户每天/每周的平均会话数

**会话时长**
每次会话的平均时长

### 留存曲线

**第 1 天留存：** 次日返回的用户百分比
**第 7 天留存：** 注册后 7 天活跃的用户百分比
**第 30 天留存：** 注册后 30 天活跃的用户百分比

**基准（第 30 天）：**

- > 40% = 优秀
- 25-40% = 良好
- < 25% = 弱

**留存曲线形状：**

- 趋于平缓 = 好（用户养成习惯）
- 急剧下降 = 产品市场匹配度差

### 病毒系数（K 因子）

```
K 因子 = 每用户邀请数 × 邀请转化率
```

**示例：**
10 次邀请/用户 × 20% 转化率 = 2.0 K 因子

**基准：**

- K > 1.0 = 病毒式增长
- K = 0.5-1.0 = 强推荐
- K < 0.5 = 弱病毒性

## B2B 指标

### 销售效率

**赢单率**

```
赢单率 = 赢单数 / 总商机数
```

**目标：** 新销售团队 20-30%，成熟团队 30-40%

**销售周期长度**
从商机到成交的平均天数

**越短越好：**

- 中小企业：30-60 天
- 中端市场：60-120 天
- 企业：120-270 天

**平均合同价值（ACV）**

```
ACV = 总合同价值 / 合同年限
```

### 管道指标

**管道覆盖率**

```
管道覆盖率 = 总管道价值 / 配额
```

**目标：** 3-5 倍覆盖（需要 3-5 倍管道才能完成配额）

**各阶段转化率：**

- 线索 → 商机：10-20%
- 商机 → 演示：50-70%
- 演示 → 提案：30-50%
- 提案 → 成交：20-40%

## 按阶段指标

### 种子前（产品市场匹配）

**关注指标：**

1. 活跃用户增长
2. 用户留存（第 7 天、第 30 天）
3. 核心参与度（会话、使用的功能）
4. 定性反馈（NPS、访谈）

**不用担心：**

- 收入（可能为零）
- CAC（尚未优化）
- 单位经济性

### 种子轮（$500K-$2M ARR）

**关注指标：**

1. MRR 增长率（15-20% 环比）
2. CAC 和 LTV（建立基线）
3. 毛留存（> 85%）
4. 核心产品参与度

**开始跟踪：**

- 销售效率
- 烧钱率和跑道

### A 轮（$2M-$10M ARR）

**关注指标：**

1. ARR 增长（3-5 倍同比）
2. 单位经济性（LTV:CAC > 3，回收期 < 18 个月）
3. 净收入留存（> 100%）
4. 烧钱倍数（< 2.0）
5. 魔法数字（> 0.5）

**成熟跟踪：**

- 40 法则
- 销售效率
- 管道覆盖率

## 指标跟踪最佳实践

### 数据基础设施

**要求：**

- 单一事实来源（分析平台）
- 实时或每日更新
- 自动计算
- 历史跟踪

**工具：**

- Mixpanel、Amplitude（产品分析）
- ChartMogul、Baremetrics（SaaS 指标）
- Looker、Tableau（BI 仪表板）

### 报告节奏

**每日：**

- MRR、活跃用户
- 注册、转化

**每周：**

- 增长率
- 留存队列
- 销售管道

**每月：**

- 完整指标套件
- 董事会报告
- 投资者更新

**每季度：**

- 趋势分析
- 对标
- 战略审查

### 常见错误

**错误 1：虚荣指标**
不要关注：

- 总用户数（不含留存）
- 页面浏览量（不含参与度）
- 下载量（不含激活）

关注与价值相关的可操作指标。

**错误 2：指标过多**
深入跟踪 5-7 个核心指标，而非松散跟踪 50 个。

**错误 3：忽视单位经济性**
即使在种子阶段，CAC 和 LTV 也至关重要。

**错误 4：不细分**
按客户群体、渠道、队列分解指标。

**错误 5：指标游戏化**
优化真正的业务成果，而非仪表板数字。

## 投资者指标

### 风投想看什么

**种子轮：**

- MRR 增长率
- 用户留存
- 早期单位经济性
- 产品参与度

**A 轮：**

- ARR 和增长率
- CAC 回收期 < 18 个月
- LTV:CAC > 3.0
- 净收入留存 > 100%
- 烧钱倍数 < 2.0

**B 轮以上：**

- 40 法则 > 40%
- 高效增长（魔法数字）
- 盈利路径
- 市场领导力指标

### 指标展示

**仪表板格式：**

```
当前 MRR：$250K（↑ 18% 环比）
ARR：$3.0M（↑ 280% 同比）
CAC：$1,200 | LTV：$4,800 | LTV:CAC = 4.0 倍
NDR：112% | 客户留存：92%
烧钱：$180K/月 | 跑道：18 个月
```

**包含：**

- 当前值
- 增长率或趋势
- 背景（目标、基准）


## 快速开始

实施创业公司指标框架：

1. **识别商业模式** - SaaS、市场、消费、B2B
2. **选择 5-7 个核心指标** - 基于阶段和模式
3. **建立跟踪** - 设置分析和仪表板
4. **计算单位经济性** - CAC、LTV、回收期
5. **设定目标** - 使用基准作为目标
6. **定期审查** - 核心指标每周
7. **与团队分享** - 在目标和进展上保持一致
8. **更新投资者** - 每月/每季度报告
