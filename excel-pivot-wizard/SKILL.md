# Excel 透视表向导

## 目录
- [概述](#概述) — [先决条件](#先决条件) — [指令](#指令) — [输出](#输出) — [示例](#示例) — [错误处理](#错误处理) — [资源](#资源)

## 概述

使用自然语言命令从原始数据创建数据透视表和可视化。自动执行从数据检查到格式化数据透视表的完整工作流，
包含图表、计算字段和条件格式，使分析师无需手动 Excel 工作即可生成精炼的报告。

## 先决条件

- Node.js 18+
- 已配置 `@negokaz/excel-mcp-server` MCP 服务器
- Claude Code 1.0+
- 本地可访问的 `.xlsx` 格式源数据

## 指令

### 步骤 1：理解数据

读取源 Excel 文件并识别：
- 列名和数据类型
- 数据粒度（交易级别、每日汇总等）
- 哪些字段是维度（分组依据）vs 度量（聚合）
- 行数和日期范围

如果不清晰，使用 AskUserQuestion 澄清每列代表什么、哪些字段应聚合 vs 分组、以及使用什么聚合函数（求和、平均、计数）。

### 步骤 2：解读请求

将自然语言解析为数据透视表结构：

| 请求 | 行 | 列 | 值 |
|---------|------|---------|--------|
| "按区域显示销售" | 区域 | -- | 销售总和 |
| "按区域和月份的销售" | 区域 | 月份 | 销售总和 |
| "按细分的平均订单价值" | 细分 | -- | 订单价值平均 |
| "按类别和代表计数订单" | 类别 | 销售代表 | 订单计数 |

### 步骤 3：构建数据透视表

使用 Excel MCP 服务器：
1. 在新工作表上创建数据透视表
2. 设置行字段（一个或多个维度）
3. 如果需要交叉表则设置列字段
4. 设置值字段及正确的聚合（求和、平均、计数、最小、最大）
5. 添加小计和总计
6. 默认按从大到小排序（日期按时间顺序）

### 步骤 4：添加增强

**计算字段**（如适用）：利润率 %、增长率 %、占总计百分比。

**条件格式：** 前 10% 深绿色，后 10% 深红色，热力图颜色渐变。

**排序：** 默认从大到小，日期字段按时间顺序。

### 步骤 5：创建可视化

根据分析模式选择图表类型：

| 模式 | 图表类型 |
|---------|-----------|
| 跨类别比较 | 柱状图/条形图 |
| 随时间趋势 | 折线图 |
| 组成/份额 | 饼图/环形图 |
| 相关性 | 散点图 |
| 多指标 | 组合图（柱状 + 折线） |

### 步骤 6：专业格式化

- 货币：$1,250,000 或 $1.25M
- 计数：千位分隔符（2,000）
- 百分比：1 位小数（35.0%）
- 粗体标题，冻结首行和首列
- 交替行颜色提高可读性

### 步骤 7：返回结果

报告数据透视表显示内容的摘要、前 3-5 个关键洞察、突出发现的具体数字、建议的后续分析，并提供添加切片器、筛选器或钻取的选项。

## 输出

- `.xlsx` 文件，在源工作簿中添加新的数据透视表工作表
- 匹配分析模式的图表可视化
- 包含关键洞察和关键数字的摘要文本
- 后续分析建议

## 示例

### 单维度汇总

```
用户："按区域显示总销售"

输出：
| 区域    | 总销售 |
|-----------|-------------|
| 西部      | $1,450,000  |
| 东北 | $1,250,000  |
| 中西部   | $1,100,000  |
| 东南 | $980,000    |
| 合计     | $4,780,000  |

洞察：西部领先，占总销售的 30.3%。
```

### 交叉表

```
用户："按区域和产品类别的销售"

输出：4x3 网格，包含行/列总计。
洞察：西部 + 电子产品 = 最高单元格 $550K。
```

## 错误处理

| 场景 | 响应 |
|----------|----------|
| 未找到数字列 | 询问用户要聚合哪个字段 |
| 模糊的维度字段 | 列出选项并请用户选择 |
| 类别太多（>50 行） | 建议 Top N 或分组方法 |
| 关键列中缺失数据 | 报告缺失百分比，提供排除空值选项 |
| 未识别日期字段 | 请用户确认日期列和格式 |

## 边缘情况

- 如果数据没有标题，从内容推断或询问用户
- 如果用户要求"分解"但未指定指标，默认为计数
- 如果存在多个数字列，询问要聚合哪个
- 如果数据跨越多个工作表，询问要分析哪个工作表
