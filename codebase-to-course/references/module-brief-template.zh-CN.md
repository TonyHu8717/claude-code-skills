# 模块简报模板

在阶段 2.5 用于复杂代码库。将完成的简报保存到 `course-name/briefs/0N-slug.md`。简报给写作代理生产一个模块所需的一切——无需阅读代码库或 SKILL.md。

---

## 模块 N：[标题]

### 1. 教学弧线

**隐喻：**
一个新鲜的、具体的隐喻——绝不能是"餐厅"。选择对这个概念感觉不可避免的东西。

**开场钩子：**
一句话，连接到学习者从使用应用已经知道的东西。

**关键见解：**
这个模块最重要的收获。一周后学习者应该记住什么？

**"为什么我应该关心？"：**
这些知识如何帮助学习者指挥 AI、调试问题或做出更好的决策？

### 2. 代码片段（预提取）

从代码库中复制粘贴实际代码。包含文件路径和行号。写作代理将逐字使用这些——它不会重新阅读代码库。

```javascript
// 文件：src/api/handler.js，第 12-24 行
const response = await fetch(url, {
  method: 'POST',
  headers: { 'Authorization': apiKey }
});
```

### 3. 互动元素

这个模块的互动元素检查清单：

- [ ] **代码↔英文对照** — 哪些片段？
- [ ] **测验** — 多少问题？什么风格？（多项选择 / 场景 / 拖放匹配 / 找 bug）
- [ ] **群聊动画** — 哪些角色？什么对话？
- [ ] **数据流动画** — 哪些角色？什么步骤？
- [ ] **拖放** — 哪些项目和目标？
- [ ] **其他** — 架构图？层切换？模式卡片？

### 4. 需要阅读的参考文件

列出写作代理需要的参考文件的具体部分：

- `references/content-philosophy.md` — 始终包含
- `references/gotchas.md` — 始终包含
- `references/interactive-elements.md` — 上述元素类型的部分
- `references/design-system.md` — 使用的视觉元素的部分
