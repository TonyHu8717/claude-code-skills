# 课程学习助手

将任意领域新手培训到前20%理论水平的交互式学习系统。

## 核心原理

### 学习目标分级
- **前20%水平**：掌握核心理论、能独立分析问题、能进行跨领域联想
- **学习金字塔**：从被动学习（听/读10%）→主动学习（讨论50%→实践70%→教授90%）

### 学习路径设计原则
1. **渐进式**：先建立全局观，再深入细节
2. **交叉验证**：同一概念多角度呈现
3. **间隔重复**：利用遗忘曲线强化记忆
4. **输出驱动**：以教促学，主动输出

## 工作流程

### Step 0: 检查已有内容

```bash
# 检查知识学习目录下是否有对应领域
ls "D:/claude/知识学习/"

# 如果有，读取现有内容了解当前进度
```

### Step 1: 领域分析

分析领域的：
- **核心概念**：该领域的基石理论（3-5个）
- **知识拓扑**：概念间的关联关系
- **学习曲线**：哪些是入门难点
- **最佳资源**：业界公认的学习路径（官方文档、经典教材、权威课程）

### Step 2: 设计学习路径

参考以下业界标准：
- **计算机领域**：Stanford/MIT 课程体系、Roadmap.sh
- **数学领域**：Khan Academy、Sullivan
- **商业领域**：MBA 核心课程、CFA/CPA 认证路径
- **通用**：Bloom's Taxonomy 六层次

学习路径结构：
```
领域/
├── 01_入门基础/        # 扫盲阶段，建立全局观
├── 02_核心概念/        # 吃透核心理论
├── 03_进阶专题/        # 深入细分领域
├── 04_实践项目/        # 实战锻炼
├── 05_综合应用/        # 融会贯通
└── 学习进度.json       # 记录学习状态
```

### Step 3: 生成交互教程

每个教程包含：
- **前置知识**：学习本节前需要掌握的内容
- **概念讲解**：通俗语言 + 比喻 + 实例
- **可视化**：图表、流程图、代码演示
- **练习题**：主动回忆 + 理解检测
- **知识检查点**：确认进入下一节的准备度
- **延伸阅读**：推荐资源

教程格式：
```markdown
# 章节标题

## 学习目标
- 理解 XXX 概念
- 掌握 XXX 方法
- 能够 XXX 应用

## 概念讲解
[通俗易懂的概念说明]

## 可视化示例
[图表或代码演示]

## 自测题
[3-5道理解检测题]

## 延伸资源
- 资源1
- 资源2
```

### Step 4: 学习进度跟踪

```json
{

  "当前阶段": "02_核心概念",

  "理解度评分": 75,
  "下一节推荐": "02_核心概念/监督学习基础"

```

## 使用方式

```
/课程学习助手 机器学习
/课程学习助手 学习 微积分
/课程学习助手 培训 量子计算
```

## 输出质量标准

- **通俗性**：高中生能理解的程度
- **系统性**：知识体系完整无遗漏
- **渐进性**：符合认知规律，步步深入
- **实用性**：理论联系实际，有案例支撑
- **可验证**：有练习题检验学习效果

## 自适应学习

根据用户反馈动态调整：
- **理解度低**：返回更基础解释，增加更多比喻
- **进度太快**：增加综合测试确保扎实
- **兴趣偏离**：询问具体想深化的方向
